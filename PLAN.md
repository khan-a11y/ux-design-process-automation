## 비전 (한 줄)
**초급 디자이너가 리더 없이 UX 프로세스(P1→P7)를 처음부터 끝까지 직접 완주하며 "왜·무엇을·어떤 맥락으로"를 체득하는 템플릿형 학습 도구.** 동시에 실무 투입까지 고도화할 MVP.

## 목적 (우선순위)
1. **(1순위) 학습** — 프로세스 경험 없는 초급 디자이너가 리더 없이 전 과정을 빠르게 "해보며" 배움. 템플릿처럼 하나씩 + 모르면 그 자리서 바로 물음.
2. **(2순위) 실무 MVP** — 각 도구가 제대로 작동 + 유기적 연동되어 실제 프로세스에 투입 가능한 수준.

## 심층 리뷰 반영 (2026-06-19)

외부 비판적 검토(`deep-research-report.md`) + **UX 디자이너·AX 전문가 관점**을 통합. 핵심 진단:

> **"지금 버전은 'UX 프로세스를 자동화하는 도구'라기보다, 'UX 프로세스를 그럴듯하게 계속 진행하게 만드는 도구'에 가깝다."**

진짜 문제 = 기능 부족이 아니라 **완성된 것처럼 느껴지는 것.** 화면 연결·단계·저장은 되지만 UX의 본질(문제 정의·근거 추적·단계 게이트·인간 검수·이용 판단 기준)이 빠짐.

### 진단 7 (요약)
**UX 관점**
1. **선형 파이프라인처럼 보임** — 반복·멈춤·되돌아가기가 없음. "다음으로"만 있고 "여기서 멈춰야 하나/가설이 틀렸나"가 없음.
2. **학습 ↔ 실무 동시추구로 둘 다 어중간** — 초보엔 과하고 숙련엔 부족.
3. **P0 문제정의가 늦게 붙음(v1.4)·재정의 루프 없음** — 초급 최대 실패는 *틀린 문제 풀기*. 리서치 후 P0 재정의가 안 됨.
4. **근거→인사이트→결정 사슬이 안 보임** — 보관함이 MD 덤프라 *이 문장이 어느 인터뷰서 나왔나*가 끊김.

**AX 관점**
5. **프롬프트가 7파일에 복제 → 드리프트** — 평문이라 `UXAX.PROMPTS`로 중앙화 가능(미실행). 개선이 한 번에 전파돼야 함.
6. **eval 부재 → 지속 향상 불가(North Star #2)** — 검증이 "렌더 되나"(smoke)뿐, "분류기가 잘 분류하나"는 측정 안 함. 측정 못 하는 건 개선 못 함.
7. **무료/로컬 메시지 vs 실제 비용 괴리** — Ollama 설치·WebGPU·모델 다운로드·한국어 전사 한계가 숨음. 단계별 정직 표기 필요.
   - (긍정) **LLM을 파이프라인 한 단계로만 쓴 설계는 정답** — 전사·군집·통계·diff는 브라우저 코드. 환각 표면적 축소. 유지.

### 확정 결정 (2026-06-19 사용자 객관식)
1. **1단계 = P1 마무리에 구조 흡수** — 남은 P1(설문/VOC/데스크/가이드/보관함)을 끝내되 **근거 메타·품질 플래그·프롬프트 중앙화**를 함께 넣어 끝냄. (현 모멘텀 유지 + 리포트 핵심 흡수)
2. **모드 = 점진적 노출** — 학습/실무 모드 토글 없이 **한 화면 레이어**(초보=설명·예제 펼침, 숙련=접고 근거·합격선만). 7파일×2 드리프트 회피.

**왜 게이트가 1단계 아닌 2단계?** 리포트는 게이트를 최우선으로 보나, 게이트·근거메타가 올라탈 **dossier 슬롯 기반이 P1 라운드(진행분)**임. 1단계에서 그 기반을 근거 메타까지 포함해 끝내고, 2단계에서 바로 게이트를 얹는 게 헛작업 없이 가장 빠름.

### 방법론 인터랙션 레이어 (3단계에 포함 — 2026-06-19 기록)
사용자 지적: 어피니티·여정맵 등 UX 방법론을 *직접 하게* 하는 도구가 빠짐. 현 어피니티 탭(P2)은 AI 자동군집 '1차안'만 주고, 실제 재배치(포스트잇 옮기기)는 도구 밖 — **탭 안내문 자체가 "사람이 항목을 옮겨 재배치하는 워크숍 전 출발점으로 쓰세요"라고 적음** = 심층리뷰의 "기능 패널" 지적 그 자리.
- **모델:** AI가 1차 초안 → 디자이너가 보드에서 끌어 재배치/병합/명명 → AI가 보조(겉도는 항목·빠진 테마 제안). = "AI 제안, 사람이 결정"(AX 원칙).
- **재사용:** "방법론 캔버스"(드래그 카드 보드) 1개로 어피니티(P2)·카드소트/IA(P3)·여정맵(P2)·우선순위 매트릭스(P3) 공용. 무빌드·`file://` 클라이언트로 구현 가능(기술 리스크 낮음, 제약은 범위).
- **배치:** 3단계(P2~P3)에 넣음. 1단계엔 안 끼움(범위 터짐 방지).

### 합격선 2단
- **동작 합격선**: 빈화면 0 · 콘솔에러 0 · smoke 통과 · 새 기능 클릭 동작.
- **고도화 합격선**: 초급자가 막힘없이 완주 + 근거 추적 + eval 기준선.

## 1단계 상세 체크리스트 — 작게 끊어서

> 한 번에 7개 HTML 갈아엎지 말 것. `common.js`의 dossier/slot/mentor/PROMPTS를 먼저 깊게 만든 뒤 각 앱을 얇게 연결. 파일 1개 수정 후 `npm test` → 빈화면/콘솔에러 하나라도 생기면 확산 중단.

### 1-0. 공통 기반 보강 (common.js 먼저)
- **슬롯 스키마 확장:** `meta`에 `source_id, source_type(interview|survey|voc|desk), confidence(0~1|상/중/하), human_confirmed(bool), used_in_phase[]` 추가. `setSlot`이 메타 기본값 채우도록.
- **프롬프트 중앙화:** 7파일에 흩어진 LLM 프롬프트를 `UXAX.PROMPTS.{task}`로 모음(평문이라 외부화 가능). 각 앱은 참조만.
- 합격선: 헤드리스 dossier read/write + 메타 저장·복원, 콘솔0.

### 1-1. Step 4 · 설문 탭 (`#survey`)
- 이름 "설문 코딩"→**"설문 자유응답 자동 분류"** + 자기설명 1줄. 전사결과/슬롯 가져오기 + CSV 만드는 법 1줄. ▶예제 **내용 미리보기**(12건 다). 응답/ID열 **미리보기 표 전건**. 코딩 설정(코드북·기준·배치) **인라인 설명·추천값**. 결과+근거메타 → `P1.survey`.

### 1-2. Step 5 · VOC/데스크/가이드 (`#voc/#desk/#guide`)
- **VOC:** "군집 분석" 쉬운 설명 + 불러온 12건 확인 + **품질 플래그**. → `P1.voc`.
- **데스크:** ▶예제 **누적**(현 덮어쓰기), 일괄 업로드, "참고 구절 수/[S1]" 쉬운 설명, 작업별 모델 배치, 멀티인풋 자동높이. → `P1.desk`.
- **가이드:** ▶예제 매번 다른 풀, "유도질문·편향 점검"을 **버튼 기능**(AI 검수)으로. → `P1.guide`.

### 1-3. Step 6 · 보관함 템플릿 뷰 + 단계 네비
- `WorkspaceModal` 재설계: dossier 슬롯을 **단계별 카드**(어디서 만듦/어디 쓰임/상태/근거메타)로, MD 덤프 대신 구조화 보기+편집. 기존 평문은 "기타 메모".
- 단계 네비: "다음 단계" 하단 이동 + 같은 depth(이전/다음 탭) + 이전 단계 돌아가기.
- 합격선: 템플릿 뷰·네비 + 새로고침 후 슬롯 유지 + **smoke 28/28 회귀**.
